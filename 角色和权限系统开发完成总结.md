## 🏗️ 系统架构

### RBAC模型
```
员工(Employee) → 角色(Role) → 权限(Permission)
     1              1              N
     ↓              ↓              ↓
   多对一         多对多         原子权限
```

### 三层权限控制
1. **角色层** - 定义角色及其等级（8个预定义角色）
2. **权限层** - 定义细粒度权限（31个预定义权限）
3. **关联层** - 角色与权限的多对多关系（role_permissions表）

```javascript

  foreignKey: 'roleId',
  as: 'employees'
});
```

### 3. Role ↔ Permission（多对多）
```javascript
Role.belongsToMany(Permission, {
  through: 'role_permissions',
  foreignKey: 'roleId',
  otherKey: 'permissionId',
  as: 'permissionDetails'
});

  through: 'role_permissions',
  foreignKey: 'permissionId',
  otherKey: 'roleId',

});
```

### 2. 基本使用示例

#### 创建自定义角色
```javascript
const role = await Role.create({
  name: '区域经理',
  code: 'REGIONAL_MANAGER',
  description: '管理区域内的销售团队',
  permissions: ['client:read', 'client:update', 'call:read'],
  level: 65,
  isActive: true,
  isSystem: false
});
```

#### 检查权限
```javascript
const role = await Role.findByCode('SALES');
if (role.hasPermission('client:read')) {
  console.log('有权限查看客户');
}
```

#### 权限中间件
```javascript
const checkPermission = (requiredPerm) => {
  return async (req, res, next) => {
    const employee = await Employee.findByPk(req.user.id, {
      include: [{ model: Role, as: 'role' }]
    });

    if (!employee?.role?.hasPermission(requiredPerm)) {
      return res.status(403).json({ error: '权限不足' });
    }
    next();
  };
};

// 使用
router.post('/clients', 
  authenticate, 
  checkPermission('client:create'), 
  createClient
);
```

## 🎨 权限设计模式

### 1. 权限命名规范
```
格式：module:action
示例：
  - client:read      (查看客户)
  - client:create    (创建客户)
  - call:update      (编辑外呼)
  - employee:delete  (删除员工)
  - *                (所有权限)
```

### 2. 模块划分
- `client` - 客户管理
- `call` - 外呼管理
- `employee` - 员工管理
- `role` - 角色管理
- `permission` - 权限管理
- `assignment` - 分配管理
- `report` - 报表管理
- `dashboard` - 仪表盘
- `system` - 系统管理

### 3. 动作定义
- `read` - 查看/读取
- `create` - 创建/新增
- `update` - 更新/编辑
- `delete` - 删除
- `export` - 导出
- `import` - 导入
- `manage` - 管理
- `view` - 查看
- `execute` - 执行

## ⚠️ 注意事项

### 1. 系统数据保护
- ⚠️ 系统角色（`isSystem: true`）不可删除
- ⚠️ 系统权限（`isSystem: true`）不可删除
- ⚠️ 修改系统数据需谨慎

### 2. 权限代码规范
- ⚠️ 格式必须是 `module:action`
- ⚠️ 只能包含小写字母、下划线和冒号
- ⚠️ 建议使用统一的命名规范

### 3. 角色等级管理
- ⚠️ 等级范围：1-100
- ⚠️ 数字越大权限越高
- ⚠️ 预留等级空间便于扩展

### 4. 性能考虑
- ⚠️ 使用JSON字段快速检查权限
- ⚠️ 定期同步permissions字段
- ⚠️ 考虑使用Redis缓存权限

### 5. 数据一致性
- ⚠️ 删除角色时员工roleId设为NULL
- ⚠️ 删除权限时自动删除关联
