# Phase 3 安装和启动指南

## 前置条件
- Node.js 16+ 已安装
- PostgreSQL 数据库已配置
- 后端服务器已配置

## 安装步骤

### 1. 安装客户端依赖

Phase 3 新增了以下依赖包：
- `antd`: Ant Design UI组件库
- `echarts`: 数据可视化图表库
- `echarts-for-react`: ECharts的React封装
- `xlsx`: Excel导出功能

```bash
# 进入客户端目录
cd client

# 安装所有依赖
npm install

# 或者单独安装新增的依赖
npm install antd@^5.22.6 echarts@^5.5.1 echarts-for-react@^3.0.2
```

### 2. 验证依赖安装

```bash
# 检查是否安装成功
npm list antd echarts echarts-for-react xlsx
```

应该看到类似输出：
```
client@0.0.0
├── antd@5.22.6
├── echarts@5.5.1
├── echarts-for-react@3.0.2
└── xlsx@0.18.5
```

### 3. 启动开发服务器

#### 启动后端服务器
```bash
# 在项目根目录
cd d:/tax-crm-system
npm start
```

后端服务器应该运行在 `http://localhost:3000`

#### 启动前端开发服务器
```bash
# 在客户端目录
cd client
npm run dev
```

前端服务器应该运行在 `http://localhost:5173`

### 4. 访问数据分析模块

1. 打开浏览器访问 `http://localhost:5173`
2. 使用管理员或经理账号登录
3. 点击左侧菜单的"数据分析"（带有"新"标签）
4. 选择以下任一分析模块：
   - 销售分析
   - 客户分析
   - 跟进分析
   - 外呼分析

## 常见问题

### 1. 依赖安装失败
**问题**: `npm install` 失败

**解决方案**:
```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 2. 图表不显示
**问题**: 页面加载但图表区域空白

**可能原因**:
- ECharts未正确安装
- 数据格式错误
- 后端API返回错误

**解决方案**:
```bash
# 检查浏览器控制台错误
# 检查网络请求是否成功
# 验证后端API返回的数据格式
```

### 3. 导出功能不工作
**问题**: 点击导出按钮没有反应

**可能原因**:
- xlsx库未安装
- 浏览器阻止下载

**解决方案**:
```bash
# 确认xlsx已安装
npm list xlsx

# 检查浏览器下载设置
# 允许弹出窗口和下载
```

### 4. 权限错误
**问题**: 无法访问数据分析页面

**解决方案**:
- 确保使用管理员(admin)或经理(manager)账号登录
- 其他角色无权访问数据分析模块

### 5. 数据库查询错误
**问题**: 后端返回500错误

**可能原因**:
- PostgreSQL语法问题
- 字段名不匹配

**解决方案**:
- 检查后端日志
- 确认数据库字段使用snake_case命名
- 确认使用PostgreSQL的DATE_TRUNC函数

## 数据库准备

确保数据库中有足够的测试数据：

```sql
-- 检查商机数据
SELECT COUNT(*) FROM "SalesOpportunities";

-- 检查客户数据
SELECT COUNT(*) FROM "Clients";

-- 检查跟进记录
SELECT COUNT(*) FROM "FollowUps";

-- 检查外呼记录
SELECT COUNT(*) FROM "CallRecords";
```

如果数据不足，图表可能显示为空。建议至少有：
- 10+ 条商机记录
- 20+ 条客户记录
- 30+ 条跟进记录
- 50+ 条外呼记录
