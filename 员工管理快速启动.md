# 🚀 员工管理模块 - 快速启动指南

## ⚡ 3分钟快速部署

### 步骤 1：运行设置脚本（1分钟）

在项目根目录运行：

```bash
node scripts/setup-employee-module.js
```

这会自动：
- ✅ 创建 employees 表
- ✅ 创建索引和触发器
- ✅ 初始化管理员账号
- ✅ 创建测试数据

### 步骤 2：重启后端服务（30秒）

```bash
# 停止当前服务（Ctrl + C）
# 重新启动
npm run dev
```

### 步骤 3：访问系统（30秒）

1. 打开浏览器：`http://localhost:5173`
2. 使用管理员账号登录：
   - 用户名：`admin`
   - 密码：`admin123`
3. 点击侧边栏"员工管理"

## ✅ 验证功能

访问员工管理页面后，你应该能看到：

- 📊 4个统计卡片（员工总数、在职、停用、离职）
- 📋 员工列表（包含管理员和3个测试员工）
- 🔍 搜索和筛选功能
- ➕ 新增员工按钮

## 🎯 核心功能测试

### 1. 查看员工列表
- 默认显示4个员工（1个管理员 + 3个测试员工）
- 可以看到工号、姓名、部门、职位等信息

### 2. 新增员工
- 点击"新增员工"按钮
- 填写必填项：姓名、用户名
- 工号和密码可留空（自动生成）
- 点击"创建员工"

### 3. 查看详情
- 点击员工列表中的"查看"图标
- 查看完整的员工信息

### 4. 编辑员工
- 点击"编辑"图标
- 修改员工信息
- 保存修改

### 5. 重置密码
- 点击"更多" > "重置密码"
- 系统会生成新密码并显示

### 6. 修改状态
- 点击"更多" > "暂停账号"或"激活账号"
- 修改员工的工作状态

## 🔧 如果遇到问题

### 问题1：表已存在
```bash
# 删除旧表
psql -U postgres -d tax_crm -c "DROP TABLE IF EXISTS employees CASCADE;"
# 重新运行设置脚本
node scripts/setup-employee-module.js
```

### 问题2：外键约束错误
确保先创建了 departments 和 roles 表：
```bash
# 检查表是否存在
psql -U postgres -d tax_crm -c "\dt"
```

### 问题3：前端404错误
检查 `client/src/App.jsx` 是否包含员工路由配置。

### 问题4：API请求失败
1. 检查后端是否正常运行（http://localhost:3000/health）
2. 检查浏览器控制台的错误信息
3. 检查后端终端的日志输出
