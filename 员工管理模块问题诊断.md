## 🔍 详细诊断

### 问题 1：500 错误 - 统计数据接口

**可能原因：**
- employees 表不存在
- Department 或 Role 模型关联问题
- 数据库连接问题

**解决方案：**

1. 检查 employees 表是否存在：
```sql
-- 在 PostgreSQL 中运行
\dt employees
```

2. 如果表不存在，运行：
```bash
node scripts/setup-employee-module.js
```

3. 检查后端日志，查看具体错误信息

### 问题 2：404 错误 - 路由不存在

**可能原因：**
- 后端服务没有重启
- 路由文件有语法错误
- 控制器方法不存在

**解决方案：**

1. **重启后端服务**（最常见原因）
```bash
# 停止服务（Ctrl + C）
npm run dev
```

2. **检查路由文件**
```bash
# 确认文件存在
ls routes/employees.js
```

3. **检查控制器方法**
```bash
# 确认所有方法都存在
grep -E "getStatistics|getDepartments|getPositions|getRoles|importEmployees" controllers/employeeController.js
```

## 🔧 常见问题解决

### Q1: 重启后还是 404？

**检查 app.js 是否注册了路由：**

```javascript
// app.js 中应该有这一行
app.use('/api/employees', employeeRoutes);
```

### Q2: 500 错误持续出现？

**查看后端终端的详细错误信息：**

```bash
# 后端终端会显示具体的错误堆栈
# 常见错误：
# - "relation 'employees' does not exist" -> 需要创建表
# - "column 'xxx' does not exist" -> 表结构不匹配
# - "Cannot read property 'xxx' of undefined" -> 关联查询问题
```

### Q3: 数据库表已存在但还是报错？

**可能是表结构不匹配，重新创建表：**

```sql
-- 删除旧表
DROP TABLE IF EXISTS employees CASCADE;
```

```bash
# 重新创建
node scripts/setup-employee-module.js
```

### Q4: 前端 Ant Design 警告？

**修复 Alert 和 Space 组件的警告：**

这些是非关键警告，不影响功能。如需修复：

```javascript
// Alert: 将 message 改为 title
<Alert title="导入说明" ... />

// Space: 将 direction 改为 orientation
<Space orientation="vertical" ... />
```
