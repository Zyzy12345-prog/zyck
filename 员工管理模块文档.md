## 🚀 快速开始

### 1. 安装依赖

确保已安装所有必要的依赖：

```bash
# 后端依赖
npm install bcryptjs

# 前端依赖（如果需要）
cd client
npm install dayjs
```

### 2. 数据库设置

运行设置脚本创建表和初始化数据：

```bash
node scripts/setup-employee-module.js
```

这个脚本会：
- 创建 `employees` 表
- 创建必要的索引
- 创建自动更新时间戳的触发器
- 初始化默认管理员账号和测试数据

### 3. 重启服务

```bash
# 重启后端
npm run dev

# 前端应该已经在运行
cd client
npm run dev
```

### 4. 访问系统

打开浏览器访问：`http://localhost:5173/employees`

使用默认管理员账号登录：
- 用户名：`admin`
- 密码：`admin123`

## 🎯 功能特性

### 1. 员工列表

- **分页显示**：支持自定义每页显示数量
- **搜索功能**：支持按姓名、工号、邮箱、手机号搜索
- **多维筛选**：按部门、职位、角色、状态筛选
- **统计卡片**：实时显示员工总数、在职、停用、离职数量
- **批量操作**：支持批量导入（预留接口）

### 2. 员工详情

- **完整信息展示**：基本信息、账号信息、职位信息、教育背景等
- **快速操作**：编辑、重置密码、修改状态、停用
- **关联数据**：显示部门、角色等关联信息

### 3. 员工表单

- **智能表单**：自动生成工号、默认密码
- **数据验证**：邮箱、手机号、身份证号格式验证
- **分组展示**：信息分组清晰，易于填写
- **编辑模式**：支持新增和编辑两种模式

### 4. 状态管理

- **在职（active）**：正常工作状态
- **离职（inactive）**：已离职，无法登录
- **停用（suspended）**：临时停用，可恢复

### 5. 密码管理

- **重置密码**：管理员可重置员工密码，生成随机密码
- **修改密码**：员工可自行修改密码（需验证原密码）
- **密码加密**：使用 bcrypt 加密存储

### 6. 权限控制

- **管理员**：完全访问权限
- **经理**：查看和编辑权限
- **普通员工**：仅查看自己的信息

## 🐛 常见问题

### 1. 表已存在错误

如果遇到"表已存在"错误，可以手动删除表后重新创建：

```sql
DROP TABLE IF EXISTS employees CASCADE;
```

然后重新运行设置脚本。

### 2. 外键约束错误

确保先创建了 `departments` 和 `roles` 表，因为 `employees` 表依赖这两个表。

### 3. 密码加密失败

确保安装了 `bcryptjs` 包：

```bash
npm install bcryptjs
```

### 4. 前端路由404

确保 `App.jsx` 中已正确配置员工管理相关路由。

### 5. 权限不足

确保登录用户具有相应的角色权限（admin 或 manager）。

### 自定义验证规则

在 `models/Employee.js` 的字段定义中添加 `validate` 选项：

```javascript
email: {
  type: DataTypes.STRING(100),
  validate: {
    isEmail: { msg: '邮箱格式不正确' },
    // 自定义验证
    isCompanyEmail(value) {
      if (value && !value.endsWith('@company.com')) {
        throw new Error('必须使用公司邮箱');
      }
    }
  }
}
```

### 扩展业务逻辑

在 `services/employeeService.js` 中添加新的业务方法，然后在 `controllers/employeeController.js` 中调用。
