# 客户详情页功能测试指南

## 🔍 问题诊断

### 可能的原因
1. 前端服务未启动或需要重启
2. 浏览器缓存问题
3. 路由配置未生效
4. 组件导入错误

---

## ✅ 解决步骤

### 步骤1: 重启前端服务

```powershell
# 在前端终端按 Ctrl+C 停止服务
# 然后重新启动
cd d:/tax-crm-system/client
npm run dev
```

### 步骤2: 清除浏览器缓存

1. 打开浏览器开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

或者使用快捷键：
- Chrome/Edge: `Ctrl + Shift + Delete`
- 选择"缓存的图片和文件"
- 点击"清除数据"

### 步骤3: 访问客户详情页

有两种方式访问：

**方式1: 从客户列表进入**
1. 访问 http://localhost:5173/customers
2. 点击任意客户名称（蓝色链接）
3. 应该跳转到 http://localhost:5173/customers/1（数字是客户ID）

**方式2: 直接访问URL**
```
http://localhost:5173/customers/1
```
（将1替换为实际的客户ID）

### 步骤4: 检查浏览器控制台

1. 按 F12 打开开发者工具
2. 切换到 Console 标签
3. 查看是否有错误信息
4. 如果有错误，请告诉我具体的错误内容

## 🐛 常见问题排查

### 问题1: 页面显示空白

**检查项**:
```powershell
# 1. 检查前端是否正常运行
# 应该看到类似输出：
# VITE v5.x.x  ready in xxx ms
# ➜  Local:   http://localhost:5173/
```

**解决方案**:
- 重启前端服务
- 检查是否有编译错误

### 问题2: 点击客户名称没有反应

**检查项**:
1. 打开浏览器控制台（F12）
2. 点击客户名称
3. 查看是否有错误信息

**可能原因**:
- 路由未正确配置
- 权限问题
- 组件导入错误

### 问题3: 显示404页面

**检查项**:
1. 确认URL格式正确：`/customers/1`
2. 确认客户ID存在

**解决方案**:
```powershell
# 检查数据库中是否有客户数据
psql -U postgres -d tax_crm -c "SELECT id, company_name FROM clients LIMIT 5;"
```

### 问题4: 显示"客户不存在"

**原因**: 数据库中没有该ID的客户

**解决方案**:
1. 返回客户列表页
2. 点击实际存在的客户
3. 或者先创建一个新客户

