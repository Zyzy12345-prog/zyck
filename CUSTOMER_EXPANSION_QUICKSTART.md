# 客户拓展模块 - 快速开始指南

## 🚀 5分钟快速部署

### 步骤1: 运行数据库迁移

```bash
cd d:/tax-crm-system
npx sequelize-cli db:migrate
```

**预期输出**:
```

Loaded configuration file "config\database.js".
Using environment "development".
== 20260218154528-create-customer-expansion-tables: migrating =======
== 20260218154528-create-customer-expansion-tables: migrated (0.234s)
```

### 步骤2: 重启后端服务器

```bash
# 停止当前服务器 (Ctrl+C)
npm start
```

**预期输出**:
```
🚀 服务器启动成功！
📍 服务地址: http://localhost:3000
```

### 步骤3: 访问新功能

打开浏览器，访问以下页面：

1. **客户线索**: http://localhost:5173/leads
2. **客户公海池**: http://localhost:5173/customer-pool

## 🎯 使用场景示例

### 场景1: 添加新线索

1. 访问 `/leads` 页面
2. 点击 **"新增线索"** 按钮
3. 填写表单：
   - 公司名称: 测试科技有限公司
   - 联系人: 张三
   - 电话: 13800138000
   - 来源: 官网
   - 优先级: 高
4. 点击 **"确定"**

### 场景2: 转化线索为客户

1. 在线索列表中找到目标线索
2. 点击 **"转化"** 按钮
3. 确认转化
4. 系统自动创建客户记录，复制线索信息

### 场景3: 从公海池领取客户

1. 访问 `/customer-pool` 页面
2. 查看可领取客户列表
3. 点击客户行的 **"领取"** 按钮
4. 确认领取
5. 客户自动分配给当前用户

### 场景4: 批量操作

1. 在线索列表中勾选多个线索
2. 点击 **"批量分配"** 按钮
3. 选择目标销售人员
4. 确认分配

## 🔧 常见问题

### Q1: 迁移失败怎么办？

**A**: 检查数据库连接配置，确保PostgreSQL服务正在运行。

```bash
# 检查数据库连接
psql -U postgres -d tax_crm -c "SELECT version();"
```

### Q2: 前端页面404错误？

**A**: 确保已重启前端开发服务器：

```bash
cd client
npm run dev
```

### Q3: API返回401错误？

**A**: 确保已登录，检查JWT token是否有效。

### Q4: 导航菜单没有显示新功能？

**A**: 检查用户角色权限，确保是 admin/manager/sales 角色。

### Q5: 线索转化失败？

**A**: 检查后端日志，确保客户表结构正确。

## 📈 性能优化建议

### 数据库索引

已自动创建以下索引：
- `customer_leads`: status, assigned_to, created_at
- `customer_pool`: client_id, status, entered_at
- `client_tag_relations`: (client_id, tag_id) 唯一索引

### 查询优化

- 使用分页查询，避免一次加载大量数据
- 使用 Sequelize 的 `include` 预加载关联数据
- 合理使用缓存（Redis）
