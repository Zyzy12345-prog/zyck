# 故障排除指南

## ❌ ERR_CONNECTION_REFUSED 错误

这个错误表示无法连接到服务器，通常是因为服务器没有启动或启动失败。

## 🔍 诊断步骤

### 步骤1：运行诊断脚本

```powershell
npm run check
```

这个脚本会检查：
- ✅ .env 文件是否存在
- ✅ 环境变量是否配置
- ✅ 依赖包是否安装
- ✅ 数据库连接是否正常

### 步骤2：检查服务器是否在运行

在项目目录运行：

```powershell
npm run dev
```

**如果看到错误信息，请根据错误类型处理：**

## 🐛 常见错误及解决方案

### 错误1：数据库连接失败

**错误信息：**
```
SequelizeConnectionError: connect ECONNREFUSED
或
SequelizeConnectionError: password authentication failed
```

**解决方案：**

1. **检查PostgreSQL服务是否运行**
   ```powershell
   # Windows PowerShell
   Get-Service -Name postgresql*
   ```
   
   如果服务未运行，启动它：
   ```powershell
   Start-Service postgresql-x64-14  # 版本号可能不同
   ```

2. **检查 .env 文件配置**
   ```env
   DB_HOST=localhost
   DB_PORT=5432
   DB_NAME=tax_crm
   DB_USER=postgres
   DB_PASSWORD=你的实际密码
   ```

3. **创建数据库**
   ```sql
   -- 使用 psql 或 pgAdmin
   CREATE DATABASE tax_crm;
   ```

4. **测试数据库连接**
   ```powershell
   psql -U postgres -d tax_crm
   ```

### 错误2：.env 文件不存在

**错误信息：**
```
Error: ENOENT: no such file or directory, open '.env'
```

**解决方案：**
```powershell
copy .env.example .env
# 然后编辑 .env 文件，填入正确的配置
```

### 错误3：依赖包未安装

**错误信息：**
```
Error: Cannot find module 'express'
```

**解决方案：**
```powershell
npm install
```

### 错误4：端口被占用

**错误信息：**
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决方案：**

**方法1：修改端口**
编辑 `.env` 文件：
```env
PORT=3001
```

**方法2：关闭占用端口的进程**
```powershell
# 查找占用3000端口的进程
netstat -ano | findstr :3000

# 结束进程（替换 <PID> 为实际进程ID）
taskkill /PID <PID> /F
```

### 错误5：数据库表不存在

**错误信息：**
```
relation "users" does not exist
```

**解决方案：**
```powershell
npm run db:migrate
```

## 📋 完整启动检查清单

在启动服务器前，确保：

- [ ] Node.js 已安装 (`node -v`)
- [ ] PostgreSQL 已安装并运行 (`psql --version`)
- [ ] 已运行 `npm install`
- [ ] 已创建 `.env` 文件并配置正确
- [ ] 已创建数据库 `tax_crm`
- [ ] 已运行 `npm run init`
- [ ] 已运行 `npm run db:migrate`
- [ ] 已运行 `npm run check` 且所有检查通过

## 🚀 正确的启动流程

```powershell
# 1. 进入项目目录
cd tax-crm-system

# 2. 安装依赖
npm install

# 3. 配置环境变量
copy .env.example .env
# 编辑 .env 文件

# 4. 创建数据库（在PostgreSQL中）
# CREATE DATABASE tax_crm;

# 5. 运行诊断
npm run check

# 6. 初始化目录
npm run init

# 7. 运行数据库迁移
npm run db:migrate

# 8. 启动服务器
npm run dev
```

## 💡 获取更多帮助

如果问题仍然存在：

1. **查看完整错误信息**
   - 服务器启动时的完整错误输出
   - 运行 `npm run check` 的输出

2. **检查日志**
   - 查看控制台输出的详细错误信息

3. **验证配置**
   - 确认所有环境变量都正确设置
   - 确认数据库服务正在运行

4. **重新开始**
   ```powershell
   # 删除并重新安装依赖
   Remove-Item -Recurse -Force node_modules
   Remove-Item package-lock.json
   npm install
   ```
