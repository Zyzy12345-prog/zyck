## 🔐 认证和权限

### 认证机制

#### JWT Token格式
```javascript
{
  employeeId: 1,
  type: 'employee',
  iat: 1234567890,
  exp: 1234567890
}
```

#### 认证流程
```
1. 登录 → 2. 获取Token → 3. 请求携带Token → 4. 验证Token → 5. 允许/拒绝
```

### 权限检查

#### 三层权限控制
1. **超级权限** (`*`) - 拥有所有权限
2. **角色权限** - 通过角色继承的权限
3. **个人权限** - 员工个人的额外权限

#### 权限中间件
- `authenticate` - JWT认证
- `checkPermission(code)` - 权限检查
- `isAdmin` - 管理员检查
- `canAccessEmployee` - 资源访问检查

## 🚀 使用示例

### 示例1：员工登录
```bash
curl -X POST http://localhost:3000/api/employees/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "password123"
  }'
```

### 示例2：创建员工
```bash
curl -X POST http://localhost:3000/api/employees \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "employeeId": "EMP002",
    "username": "zhangsan",
    "password": "password123",
    "fullName": "张三",
    "email": "zhangsan@example.com",
    "department": "销售部",
    "position": "销售经理",
    "roleId": 5
  }'
```

### 示例3：查询员工列表
```bash
curl -X GET "http://localhost:3000/api/employees?page=1&limit=10&department=销售部" \
  -H "Authorization: Bearer <token>"
```

### 示例4：分配角色
```bash
curl -X POST http://localhost:3000/api/employees/1/role \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "roleId": 2
  }'
```

## ⚠️ 注意事项

### 1. 安全性
- ⚠️ 所有密码都经过bcrypt加密
- ⚠️ Token需要妥善保管
- ⚠️ 不能删除自己的账号
- ⚠️ 管理员操作需要权限验证

### 2. 数据一致性
- ⚠️ 删除是软删除（状态改为inactive）
- ⚠️ 邮箱和用户名必须唯一
- ⚠️ 角色分配会同步权限

### 3. 性能考虑
- ⚠️ 列表查询支持分页
- ⚠️ 使用索引优化查询
- ⚠️ 关联查询使用include

- ⚠️ 返回统一的错误格式
