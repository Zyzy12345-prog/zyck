## 🎯 使用场景

### 场景1：新增单个员工

**步骤**：
1. 访问员工列表页面
2. 点击右上角"新增员工"按钮
3. 填写表单：
   - 必填：用户名、姓名、邮箱
   - 可选：工号（留空自动生成）、密码（留空自动生成）
   - 选择：部门、职位、角色
4. 点击"创建员工"
5. 记录显示的初始密码

**示例数据**：
```
用户名: zhangsan
姓名: 张三
邮箱: zhangsan@example.com
部门: 技术部
职位: 软件工程师
角色: 普通员工
```

---

### 场景2：批量导入员工

**步骤**：
1. 点击"批量导入"按钮
2. 下载导入模板
3. 填写Excel表格：
   ```
   员工工号 | 用户名* | 姓名* | 邮箱* | 手机号 | 部门 | 职位 | 状态
   EMP001  | zhangsan | 张三 | zhangsan@example.com | 13800138000 | 技术部 | 工程师 | 在职
   EMP002  | lisi     | 李四 | lisi@example.com     | 13900139000 | 市场部 | 专员   | 在职
   ```
4. 上传填写好的Excel文件
5. 查看验证结果（有效/错误/警告）
6. 点击"开始导入"
7. 等待导入完成
8. 导出导入结果（包含初始密码）

**注意事项**：
- 带 * 的字段为必填
- 用户名和邮箱必须唯一
- 文件大小不超过5MB
- 支持 .xlsx 和 .xls 格式

---

### 场景3：搜索和筛选员工

**搜索**：
- 在搜索框输入关键词
- 支持搜索：姓名、工号、邮箱、手机号
- 实时显示搜索结果

**筛选**：
- 部门筛选：选择特定部门
- 职位筛选：选择特定职位
- 状态筛选：在职/离职/停用
- 角色筛选：选择特定角色
- 组合筛选：可同时使用多个筛选条件

**重置**：
- 点击"重置"按钮清除所有筛选条件

---

### 场景4：编辑员工信息

**步骤**：
1. 在员工列表找到目标员工
2. 点击"编辑"按钮
3. 修改需要更新的字段
4. 点击"保存修改"

**限制**：
- 工号不可修改
- 用户名不可修改
- 邮箱需保持唯一性

---

### 场景5：管理员工状态

**激活账号**：
1. 点击员工行的"更多"按钮
2. 选择"激活账号"
3. 确认操作

**暂停账号**：
1. 点击"更多"按钮
2. 选择"暂停账号"
3. 确认操作

**停用员工**：
1. 点击"更多"按钮
2. 选择"停用员工"
3. 确认操作（软删除，数据保留）

---

### 场景6：重置员工密码

**步骤**：
1. 点击员工行的"更多"按钮
2. 选择"重置密码"
3. 确认操作
4. 记录新密码
5. 告知员工新密码

**注意**：
- 新密码会自动生成
- 建议员工首次登录后修改密码

## 🔑 字段说明

### 必填字段
| 字段名 | 说明 | 验证规则 |
|--------|------|----------|
| 用户名 | 登录账号 | 3-50位，字母/数字/下划线，唯一 |
| 姓名 | 员工真实姓名 | 2-100位字符 |
| 邮箱 | 联系邮箱 | 有效邮箱格式，唯一 |

### 可选字段
| 字段名 | 说明 | 验证规则 |
|--------|------|----------|
| 员工工号 | 员工编号 | 留空自动生成（年月+4位序号） |
| 密码 | 登录密码 | 最少6位，留空自动生成8位 |
| 手机号 | 联系电话 | 11位手机号 |
| 性别 | 性别 | 男/女/其他 |
| 出生日期 | 出生日期 | YYYY-MM-DD格式 |
| 身份证号 | 身份证号码 | 18位身份证号 |
| 部门 | 所属部门 | 从列表选择或输入 |
| 职位 | 职位名称 | 从列表选择或输入 |
| 职级 | 职级 | 初级/中级/高级/资深/专家 |
| 入职日期 | 入职日期 | YYYY-MM-DD格式 |
| 状态 | 员工状态 | 在职/离职/停用/已离职 |
| 角色 | 系统角色 | 从角色列表选择 |
| 是否管理员 | 管理员权限 | 是/否 |
| 备注 | 备注信息 | 最多500字符 |

## 📊 状态说明

### 员工状态
- **在职** (active)：正常工作状态，可以登录系统
- **离职** (inactive)：已离职，无法登录系统
- **停用** (suspended)：临时停用，无法登录系统
- **已离职** (resigned)：正式离职，无法登录系统

### 状态转换
```
在职 ⟷ 停用
在职 → 离职
停用 → 离职
```

## ⚠️ 常见问题

### Q1: 创建员工时提示"用户名已存在"？
**A**: 用户名必须唯一，请更换其他用户名。

### Q2: 批量导入时部分数据失败？
**A**: 
1. 检查错误信息
2. 导出错误数据
3. 修正错误后重新导入

### Q3: 忘记员工初始密码？
**A**: 使用"重置密码"功能生成新密码。

### Q4: 如何批量修改员工部门？
**A**: 目前需要逐个编辑，批量操作功能开发中。

### Q5: 导入的Excel文件解析失败？
**A**: 
1. 确保使用官方模板
2. 检查文件格式（.xlsx 或 .xls）
3. 确保文件未损坏
4. 检查文件大小（不超过5MB）

### Q6: 员工列表加载很慢？
**A**: 
1. 使用筛选功能缩小范围
2. 减少每页显示数量
3. 清除浏览器缓存

## 🔧 技术支持

### API 端点
```
GET    /api/employees          # 获取员工列表
GET    /api/employees/:id      # 获取员工详情
POST   /api/employees          # 创建员工
PUT    /api/employees/:id      # 更新员工
DELETE /api/employees/:id      # 删除员工
PATCH  /api/employees/:id/status  # 修改状态
POST   /api/employees/:id/reset-password  # 重置密码
```

## 📝 最佳实践

### 1. 新员工入职流程
```
1. 使用"新增员工"创建账号
2. 记录初始密码
3. 分配角色和权限
4. 告知员工登录信息
5. 要求首次登录后修改密码
```

### 2. 批量导入流程
```
1. 准备员工数据（Excel格式）
2. 下载官方模板
3. 按模板格式填写数据
4. 上传并验证数据
5. 确认导入
6. 导出结果（含初始密码）
7. 分发初始密码给员工
```

### 3. 员工离职流程
```
1. 修改员工状态为"离职"
2. 记录离职日期
3. 备份员工数据
4. 转移工作任务
5. 归档员工信息
```
