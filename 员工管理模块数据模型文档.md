# 员工管理模块数据模型文档

## 📋 概述

员工管理模块（Employee Model）是财税CRM系统的核心模块之一，用于管理系统内部员工的基本信息、工作状态、权限配置和业绩统计。

**文件位置**：`models/Employee.js`  
**数据库表名**：`employees`  
**创建时间**：2026-01-23

---

## 📊 数据模型结构

### 1. 员工基本信息

| 字段名 | 数据类型 | 必填 | 唯一 | 说明 |
|--------|---------|------|------|------|
| `id` | INTEGER | ✅ | ✅ | 主键，自增 |
| `employeeId` | STRING(20) | ✅ | ✅ | 员工工号（如：EMP001） |
| `username` | STRING(50) | ✅ | ✅ | 登录用户名（3-50字符，仅字母数字） |
| `password` | STRING(255) | ✅ | ❌ | 密码（bcrypt加密，最少6位） |
| `fullName` | STRING(100) | ✅ | ❌ | 员工姓名（2-100字符） |
| `email` | STRING(100) | ✅ | ✅ | 邮箱地址（必须是有效邮箱格式） |
| `phone` | STRING(20) | ❌ | ❌ | 手机号码（支持中国手机号验证） |
| `department` | STRING(100) | ❌ | ❌ | 所属部门 |
| `position` | STRING(100) | ❌ | ❌ | 职位名称 |
| `avatar` | STRING(500) | ❌ | ❌ | 头像URL地址 |

### 2. 工作信息

| 字段名 | 数据类型 | 必填 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| `status` | ENUM | ✅ | 'active' | 员工状态 |
| `hireDate` | DATEONLY | ❌ | null | 入职日期（不能晚于今天） |
| `lastLogin` | DATE | ❌ | null | 最后登录时间 |
| `loginIp` | STRING(45) | ❌ | null | 最后登录IP（支持IPv4/IPv6） |

**状态枚举值**：
- `active` - 在职
- `inactive` - 离职
- `suspended` - 停用

### 3. 权限信息

| 字段名 | 数据类型 | 必填 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| `roleId` | INTEGER | ❌ | null | 角色ID（外键关联roles表） |
| `permissions` | ARRAY(STRING) | ❌ | [] | 权限数组 |
| `accessLevel` | INTEGER | ✅ | 1 | 访问等级（1-10，数字越大权限越高） |

**权限数组示例**：
```javascript
[
  "client:read",      // 查看客户
  "client:write",     // 编辑客户
  "client:delete",    // 删除客户
  "call:read",        // 查看通话记录
  "call:write",       // 创建通话记录
  "report:view",      // 查看报表
  "*"                 // 超级权限（所有权限）
]
```

### 4. 统计信息

| 字段名 | 数据类型 | 必填 | 默认值 | 说明 |
|--------|---------|------|--------|------|
| `totalClients` | INTEGER | ✅ | 0 | 分配的客户总数 |
| `activeClients` | INTEGER | ✅ | 0 | 当前跟进客户数 |
| `completedCalls` | INTEGER | ✅ | 0 | 完成通话数 |
| `successRate` | DECIMAL(5,2) | ✅ | 0.00 | 转化成功率（0-100%） |

### 5. 时间戳

| 字段名 | 数据类型 | 必填 | 说明 |
|--------|---------|------|------|
| `createdAt` | DATE | ✅ | 创建时间（自动生成） |
| `updatedAt` | DATE | ✅ | 更新时间（自动更新） |

## 🔗 关联关系

### 1. 员工 → 角色（多对一）
```javascript
Employee.belongsTo(Role, {
  foreignKey: 'roleId',
  as: 'role'
});
```

### 2. 员工 → 分配记录（一对多）
```javascript
Employee.hasMany(Assignment, {
  foreignKey: 'userId',
  as: 'assignments'
});
```

### 3. 员工 → 通话记录（一对多）
```javascript
Employee.hasMany(CallRecord, {
  foreignKey: 'userId',
  as: 'callRecords'
});
```

### 4. 员工 ↔ 客户（多对多，通过Assignment）
```javascript
Employee.belongsToMany(Client, {
  through: Assignment,
  foreignKey: 'userId',
  otherKey: 'clientId',
  as: 'clients'
});
```

## 🔐 安全特性

### 1. 密码加密
使用 `bcrypt` 进行密码加密，盐值轮数为10：

```javascript
// 创建时自动加密
beforeCreate: async (employee) => {
  const salt = await bcrypt.genSalt(10);
  employee.password = await bcrypt.hash(employee.password, salt);
}

// 更新时自动加密
beforeUpdate: async (employee) => {
  if (employee.changed('password')) {
    const salt = await bcrypt.genSalt(10);
    employee.password = await bcrypt.hash(employee.password, salt);
  }
}
```

### 2. 密码验证
```javascript
const isMatch = await employee.comparePassword('输入的密码');
```

### 3. 隐藏敏感信息
```javascript
// toJSON 方法自动删除密码字段
const safeData = employee.toJSON(); // 不包含password
```

## 📝 实例方法

### 1. 密码相关

#### `comparePassword(candidatePassword)`
比较输入密码与存储密码是否匹配

```javascript
const employee = await Employee.findByUsername('john');
const isValid = await employee.comparePassword('password123');
if (isValid) {
  console.log('密码正确');
}
```

### 2. 登录相关

#### `updateLastLogin(ip)`
更新最后登录时间和IP

```javascript
await employee.updateLastLogin('192.168.1.100');
```

### 3. 统计相关

#### `updateStats(stats)`
更新员工统计信息

```javascript
await employee.updateStats({
  totalClients: 50,
  activeClients: 30,
  completedCalls: 200,
  successRate: 75.5
});
```

### 4. 权限相关

#### `hasPermission(permission)`
检查是否拥有特定权限

```javascript
if (employee.hasPermission('client:write')) {
  console.log('有权限编辑客户');
}
```

#### `addPermissions(permissions)`
添加权限

```javascript
await employee.addPermissions(['client:read', 'client:write']);
// 或单个权限
await employee.addPermissions('report:view');
```

#### `removePermissions(permissions)`
移除权限

```javascript
await employee.removePermissions(['client:delete']);
```

### 5. 状态相关

#### `isActive()`
检查员工是否在职

```javascript
if (employee.isActive()) {
  console.log('员工在职');
}
```

#### `getWorkDays()`
获取工作天数

```javascript
const days = employee.getWorkDays();
console.log(`已工作 ${days} 天`);
```

### 6. 数据转换

#### `toJSON()`
转换为安全的JSON对象（隐藏密码）

```javascript
const safeData = employee.toJSON();
// 返回所有字段，但不包含password
```

#### `getPublicProfile()`
获取公开信息（仅基本字段）

```javascript
const publicInfo = employee.getPublicProfile();
// 返回：{ id, employeeId, fullName, department, position, avatar, status }
```

## 📊 索引设计

### 唯一索引
- `idx_employee_id` - 员工工号
- `idx_username` - 用户名
- `idx_email` - 邮箱

### 普通索引
- `idx_status` - 状态
- `idx_department` - 部门
- `idx_role_id` - 角色ID
- `idx_hire_date` - 入职日期
- `idx_access_level` - 访问等级

### 复合索引
- `idx_status_department` - 状态+部门（常用组合查询）
- `idx_created_status` - 创建时间+状态（列表排序）

## 💡 使用示例

### 1. 创建员工

```javascript
const employee = await Employee.create({
  employeeId: 'EMP001',
  username: 'john_doe',
  password: 'password123',
  fullName: '张三',
  email: 'zhangsan@example.com',
  phone: '13800138000',
  department: '销售部',
  position: '销售经理',
  hireDate: '2024-01-15',
  roleId: 2,
  permissions: ['client:read', 'client:write', 'call:read'],
  accessLevel: 5
});
```

### 2. 员工登录

```javascript
// 查找员工
const employee = await Employee.findByUsername('john_doe');

if (!employee) {
  throw new Error('用户不存在');
}

// 验证密码
const isValid = await employee.comparePassword('password123');

if (!isValid) {
  throw new Error('密码错误');
}

// 检查状态
if (!employee.isActive()) {
  throw new Error('账号已停用');
}

// 更新登录信息
await employee.updateLastLogin('192.168.1.100');

// 返回安全数据
return employee.toJSON();
```

### 3. 更新员工信息

```javascript
const employee = await Employee.findByPk(1);

await employee.update({
  department: '市场部',
  position: '市场总监',
  accessLevel: 7
});
```

### 4. 权限管理

```javascript
const employee = await Employee.findByPk(1);

// 添加权限
await employee.addPermissions(['report:view', 'report:export']);

// 检查权限
if (employee.hasPermission('report:export')) {
  // 允许导出报表
}

// 移除权限
await employee.removePermissions(['report:export']);
```

### 5. 统计信息更新

```javascript
const employee = await Employee.findByPk(1);

// 更新业绩统计
await employee.updateStats({
  totalClients: 100,
  activeClients: 60,
  completedCalls: 500,
  successRate: 80.5
});
```

### 6. 查询员工列表

```javascript
// 获取所有在职员工
const activeEmployees = await Employee.getActiveEmployees({
  include: [
    { model: Role, as: 'role' },
    { model: Client, as: 'clients' }
  ],
  order: [['createdAt', 'DESC']]
});

// 按部门查询
const salesTeam = await Employee.getByDepartment('销售部');

// 复杂查询
const employees = await Employee.findAll({
  where: {
    status: 'active',
    accessLevel: { [Op.gte]: 5 },
    department: { [Op.in]: ['销售部', '市场部'] }
  },
  include: [{ model: Role, as: 'role' }],
  order: [['successRate', 'DESC']],
  limit: 10
});
```

### 7. 关联查询

```javascript
// 查询员工及其客户
const employee = await Employee.findByPk(1, {
  include: [
    {
      model: Client,
      as: 'clients',
      through: { attributes: [] } // 不返回中间表字段
    },
    {
      model: CallRecord,
      as: 'callRecords',
      limit: 10,
      order: [['callTime', 'DESC']]
    },
    {
      model: Role,
      as: 'role'
    }
  ]
});
```

## ⚠️ 注意事项

### 1. 数据验证
- 用户名只能包含字母和数字
- 密码最少6位
- 邮箱必须是有效格式
- 手机号必须符合中国手机号格式
- 入职日期不能晚于今天
- 访问等级必须在1-10之间
- 成功率必须在0-100之间

### 2. 数据一致性
- 活跃客户数不能超过总客户数（自动修正）
- 成功率自动限制在0-100范围内
- 删除角色时，员工的roleId会被设置为NULL

### 3. 性能优化
- 使用索引加速常用查询
- 复合索引优化组合查询
- 使用`toJSON()`避免返回敏感信息

### 4. 安全建议
- 永远不要在日志中输出密码
- 使用`toJSON()`或`getPublicProfile()`返回数据
- 定期更新密码加密算法
- 记录登录IP用于安全审计
