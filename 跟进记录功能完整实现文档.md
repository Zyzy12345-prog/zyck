```sql
CREATE TABLE IF NOT EXISTS follow_up_reminders (...);
```

### 数据表结构

#### follow_ups（跟进记录表）
```sql
- id: 主键
- client_id: 客户ID
- user_id: 跟进人ID
- follow_type: 跟进方式（phone/visit/email/wechat/meeting/other）
- follow_time: 跟进时间
- content: 跟进内容（支持富文本）
- next_follow_time: 下次跟进时间
- status: 状态（pending/completed/cancelled）
- result: 跟进结果（success/next_stage/need_follow/failed/pending/no_answer）
- attachments: 附件信息（JSON）
- is_reminded: 是否已提醒
- reminded_at: 提醒时间
```

#### follow_up_comments（跟进评论表）
```sql
- id: 主键
- follow_up_id: 跟进记录ID
- user_id: 评论人ID
- content: 评论内容
- created_at: 创建时间
```

#### follow_up_reminders（跟进提醒表）
```sql
- id: 主键
- follow_up_id: 跟进记录ID
- user_id: 提醒用户ID
- reminder_type: 提醒类型（email/system/sms）
- reminder_time: 提醒时间
- status: 状态（pending/sent/failed）
- sent_at: 发送时间
```

## 📝 使用说明

### 1. 创建跟进记录

1. 进入客户详情页
2. 切换到"跟进记录"标签
3. 点击"新增跟进"按钮
4. 填写表单：
   - 选择跟进方式
   - 选择跟进时间
   - 输入跟进内容（可选择富文本模式）
   - 选择跟进结果
   - 设置下次跟进时间（可选）
   - 上传相关文件（可选）
5. 点击"提交"

### 2. 添加评论

1. 在跟进记录卡片上点击"评论"按钮
2. 输入评论内容
3. 点击"确定"

### 3. 查看跟进日历

1. 在客户详情页切换到"跟进日历"标签
2. 查看月视图中的跟进计划
3. 点击有事件的日期查看详情
4. 点击"查看详情"跳转到客户页面

### 4. 文件管理

1. 在"相关文件"标签页查看所有文件
2. 可以按分类筛选文件
3. 点击"下载"按钮下载文件
4. 点击"删除"按钮删除文件

## 📊 数据流程

### 创建跟进记录流程

```
用户填写表单
    ↓
前端验证
    ↓
发送POST请求到 /api/clients/:clientId/follow-ups
    ↓
后端验证权限
    ↓
创建跟进记录
    ↓
如果设置了下次跟进时间
    ↓
创建提醒记录
    ↓
返回完整数据（包含用户、评论、文件）
    ↓
前端刷新列表
```

### 文件上传流程

```
用户选择文件
    ↓
前端验证文件类型和大小
    ↓
使用FormData上传
    ↓
后端multer中间件处理
    ↓
保存到 uploads/customer-files/
    ↓
创建数据库记录
    ↓
返回文件信息
```

## 🐛 常见问题

### Q1: 富文本编辑器不显示？
**A**: 确保已安装 react-quill
```bash
npm install react-quill
```

### Q2: 文件上传失败？
**A**: 检查：
1. uploads/customer-files 目录是否存在
2. 文件大小是否超过10MB
3. 文件类型是否支持

### Q3: 提醒不工作？
**A**: 提醒功能需要配置定时任务，当前仅创建提醒记录，需要额外实现发送逻辑

### Q4: 日历数据不显示？
**A**: 检查：
1. 是否有设置下次跟进时间的记录
2. 日期范围是否正确
3. 后端API是否正常
