# 员工批量导入功能 - 安装说明

## 📦 需要安装的依赖

批量导入功能需要 `xlsx` 库来解析 Excel 文件。

### 安装步骤

在前端项目目录运行：

```bash
cd client
npm install xlsx
```

## ✅ 功能说明

### 1. 批量导入流程

**步骤 1：上传文件**
- 点击"批量导入"按钮
- 下载导入模板
- 填写员工信息
- 上传 Excel 文件（支持 .xlsx 和 .xls）

**步骤 2：验证数据**
- 系统自动验证数据格式
- 显示验证结果（通过/失败/警告）
- 可以查看每条数据的错误信息

**步骤 3：导入结果**
- 显示导入成功和失败的数量
- 列出成功导入的员工（工号、姓名）
- 列出失败的员工及失败原因

### 3. 数据验证规则

**必填字段验证：**
- 姓名：不能为空，2-50个字符
- 用户名：不能为空，3-50个字符

**格式验证：**
- 邮箱：必须符合邮箱格式
- 手机号：必须是11位数字，以1开头
- 身份证号：必须是18位，符合身份证号规则

**唯一性验证：**
- 用户名：不能与现有员工重复
- 邮箱：不能与现有员工重复（如果填写）

**默认值：**
- 密码：如果不填写，默认为 123456
- 工号：系统自动生成
- 状态：默认为"在职"

### 4. 导入限制

- 文件大小：不超过 10MB
- 文件格式：.xlsx 或 .xls
- 单次导入：建议不超过 1000 条

### 5. 常见问题

**Q: 导入失败，提示"用户名已存在"？**
A: 检查 Excel 中的用户名是否与系统中已有员工重复，或者同一个 Excel 文件中是否有重复的用户名。

**Q: 导入后密码是什么？**
A: 如果 Excel 中没有填写密码，系统会使用默认密码 123456。建议员工首次登录后修改密码。

**Q: 部门和角色如何填写？**
A: 部门填写部门名称（如"销售部"），角色填写角色代码（如"SALES"、"ADMIN"等）。如果不确定，可以留空，后续在系统中编辑。

**Q: 可以更新现有员工信息吗？**
A: 当前版本只支持新增员工，不支持更新。如需更新，请使用编辑功能。

**Q: 导入失败的数据会保存吗？**
A: 不会。只有验证通过的数据才会被导入，失败的数据需要修改后重新导入。

## 📝 注意事项

1. **Excel 格式**：确保第一行是表头，数据从第二行开始
2. **字段顺序**：可以调整列的顺序，但表头名称必须与模板一致
3. **空值处理**：非必填字段可以留空
4. **日期格式**：日期必须是 YYYY-MM-DD 格式（如 2024-01-01）
5. **批量操作**：建议分批导入，每批不超过 500 条

## 🚀 快速开始

1. 安装依赖：`cd client && npm install xlsx`

4. 点击"批量导入"按钮
5. 下载模板并填写数据
6. 上传文件并完成导入
