# 员工管理模块关联问题修复报告

## 问题描述

登录成功后，访问员工管理模块时出现错误：

```
获取员工列表失败：Role is associated to Employee using an alias. 
You've included an alias (role), but it does not match the alias(es) defined in your association (roles).
```

## 问题原因

Employee 模型与 Role 模型之间是**多对多关系**（通过 EmployeeRole 中间表），在模型定义中使用的别名是 `roles`（复数），但在 `employeeController.js` 中错误地使用了 `role`（单数）。

### 模型定义（正确）

```javascript
// models/Employee.js
Employee.associate = (models) => {
  // 关联角色（多对多）
  Employee.belongsToMany(models.Role, {
    through: models.EmployeeRole,
    as: 'roles',  // ✅ 使用复数
    foreignKey: 'employeeId',
    otherKey: 'roleId'
  });
};
```

### Controller 使用（错误）

```javascript
// controllers/employeeController.js
include: [{ 
  model: Role, 
  as: 'role'  // ❌ 使用了单数，不匹配
}]
```

## 修复方案

### 1. 修复模型导出格式

首先修复了模型文件的导出格式，使其与 `models/index.js` 的加载方式兼容：

**修复的文件：**
- `models/Department.js`
- `models/Employee.js`
- `models/Role.js`
- `models/EmployeeRole.js`


## 关键改进

### 1. 支持多角色

员工现在可以拥有多个角色，权限是所有角色权限的并集：

```javascript
Employee.prototype.getAllPermissions = async function() {
  if (this.isAdmin) {
    return ['*']; // 管理员拥有所有权限
  }

  const roles = await this.getRoles();
  const allPermissions = new Set();

  roles.forEach(role => {
    const permissions = role.permissions || {};
    Object.keys(permissions).forEach(key => {
      if (permissions[key] === true) {
        allPermissions.add(key);
      }
    });
  });

  return Array.from(allPermissions);
};
```

### 2. 使用 Sequelize 自动生成的方法

多对多关系会自动生成以下方法：

```javascript
// 添加角色
await employee.addRole(role);
await employee.addRoles([role1, role2]);

// 移除角色
await employee.removeRole(role);
await employee.removeRoles([role1, role2]);

// 获取角色
const roles = await employee.getRoles();

// 设置角色（替换所有）
await employee.setRoles([role1, role2]);

// 检查是否有某个角色
const hasRole = await employee.hasRole(role);
```

### 3. 排除中间表字段

使用 `through: { attributes: [] }` 避免返回中间表的额外字段：

```javascript
include: [
  {
    model: Role,
    as: 'roles',
    through: { attributes: [] }  // 不返回 employee_roles 表的字段
  }
]
```

## 注意事项

### 1. 前端适配

前端代码需要相应调整，将 `employee.role` 改为 `employee.roles`：

```javascript
// 修改前
const roleName = employee.role?.name;

// 修改后
const roleNames = employee.roles?.map(r => r.name).join(', ');
```

### 2. 权限检查

权限检查需要遍历所有角色：

```javascript
// 检查是否有某个权限
const hasPermission = async (employee, permission) => {
  if (employee.isAdmin) return true;
  
  const roles = await employee.getRoles();
  return roles.some(role => role.permissions[permission] === true);
};
```
