# 客户详情页（360度视图）- 快速参考

## 🚀 快速启动

### 方法1：使用启动脚本（推荐）
```powershell
# 在项目根目录执行
.\start-customer-detail.ps1
```

### 方法2：手动启动
```bash
# 1. 运行数据库迁移
npx sequelize-cli db:migrate

# 2. 安装前端依赖（如果需要）
cd client
npm install dayjs
cd ..

# 3. 启动后端
npm start

# 4. 启动前端（新终端）
cd client
npm run dev
```

## 📍 访问路径

- **客户列表**: http://localhost:5173/customers
- **客户详情**: http://localhost:5173/customers/:id
- **示例**: http://localhost:5173/customers/1

## 🎯 核心功能

### 1. 客户基本信息
- 公司名称、联系人、电话、邮箱
- 公司地址、网址、规模、员工人数
- 成立日期、法定代表人、经营范围
- 客户等级（A/B/C/D）、客户来源
- 注册资本、财税状态、所属行业

### 2. 跟进记录管理
```javascript
// 创建跟进记录
POST /api/clients/:clientId/follow-ups
{
  "followType": "phone",        // phone/visit/email/wechat/meeting/other
  "followTime": "2026-02-05T10:00:00Z",
  "content": "跟进内容",
  "result": "success",          // success/failed/pending/no_answer
  "nextFollowTime": "2026-02-10T10:00:00Z"
}

// 获取跟进记录
GET /api/clients/:clientId/follow-ups?page=1&limit=20

// 删除跟进记录
DELETE /api/clients/follow-ups/:id
```

### 3. 文件管理
```javascript
// 上传文件
POST /api/clients/:clientId/files
{
  "fileName": "营业执照.pdf",
  "filePath": "/uploads/xxx.pdf",
  "fileSize": 1024000,
  "fileType": "application/pdf",
  "category": "certificate",    // contract/invoice/certificate/report/other
  "description": "文件描述"
}

// 获取文件列表
GET /api/clients/:clientId/files?category=contract

// 删除文件
DELETE /api/clients/files/:id
```

### 4. 内部讨论
```javascript
// 发表讨论
POST /api/clients/:clientId/discussions
{
  "content": "讨论内容",
  "parentId": null              // 回复时填写父评论ID
}

// 获取讨论列表
GET /api/clients/:clientId/discussions

// 删除讨论
DELETE /api/clients/discussions/:id
```

## 📊 数据字典

### 客户等级 (customer_level)
- `A` - 重点客户（红色标签）
- `B` - 优质客户（橙色标签）
- `C` - 普通客户（蓝色标签）
- `D` - 潜在客户（灰色标签）

### 公司规模 (company_scale)
- `micro` - 微型(1-9人)
- `small` - 小型(10-49人)
- `medium` - 中型(50-299人)
- `large` - 大型(300人以上)

### 财税状态 (tax_status)
- `pending` - 待处理
- `processing` - 处理中
- `completed` - 已完成
- `cancelled` - 已取消

### 跟进方式 (follow_type)
- `phone` - 电话
- `visit` - 拜访
- `email` - 邮件
- `wechat` - 微信
- `meeting` - 会议
- `other` - 其他

### 跟进结果 (result)
- `success` - 成功
- `failed` - 失败
- `pending` - 待定
- `no_answer` - 未接通

### 文件分类 (category)
- `contract` - 合同
- `invoice` - 发票
- `certificate` - 证书
- `report` - 报告
- `other` - 其他

## 🔧 常见问题

### Q1: 数据库迁移失败？
**A**: 检查数据库连接配置，确保 `config/database.js` 中的配置正确

### Q2: 页面显示空白？
**A**: 
1. 检查后端服务是否启动
2. 检查浏览器控制台是否有错误
3. 确认客户ID是否存在

### Q3: 跟进记录无法创建？
**A**: 
1. 检查用户是否有权限
2. 确认表单字段是否填写完整
3. 查看后端日志错误信息

### Q4: 文件上传功能不可用？
**A**: 文件上传功能需要配置服务器存储路径，目前仅提供接口框架

### Q5: dayjs 未安装？
**A**: 
```bash
cd client
npm install dayjs
```
