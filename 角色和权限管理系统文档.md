# 角色和权限管理系统文档

## 📋 概述

角色和权限管理系统是财税CRM系统的核心安全模块，采用RBAC（基于角色的访问控制）模型，实现灵活的权限管理。

**创建时间**：2026-01-23  
**版本**：v1.0.0

---

## 🏗️ 系统架构

### RBAC模型
```
员工(Employee) → 角色(Role) → 权限(Permission)
     1              1              N
     ↓              ↓              ↓
   多对一         多对多         原子权限
```

### 三层权限控制
1. **角色层**：定义角色及其等级
2. **权限层**：定义细粒度权限
3. **关联层**：角色与权限的多对多关系

## 📊 数据模型

### 1. Role（角色模型）

**文件位置**：`models/Role.js`  
**数据库表**：`roles`

#### 字段说明

| 字段名 | 类型 | 必填 | 唯一 | 说明 |
|--------|------|------|------|------|
| id | INTEGER | ✅ | ✅ | 主键 |
| name | STRING(50) | ✅ | ✅ | 角色名称（如：超级管理员） |
| code | STRING(50) | ✅ | ✅ | 角色代码（如：SUPER_ADMIN） |
| description | TEXT | ❌ | ❌ | 角色描述 |
| permissions | JSON | ❌ | ❌ | 权限代码数组（快速检查） |
| level | INTEGER | ✅ | ❌ | 角色等级（1-100） |
| isActive | BOOLEAN | ✅ | ❌ | 是否启用 |
| isSystem | BOOLEAN | ✅ | ❌ | 是否系统内置 |
| createdAt | DATE | ✅ | ❌ | 创建时间 |
| updatedAt | DATE | ✅ | ❌ | 更新时间 |

#### 实例方法

```javascript
// 检查权限
role.hasPermission('client:read')

// 添加权限
await role.addPermissions(['client:write', 'client:delete'])

// 移除权限
await role.removePermissions(['client:delete'])

// 同步权限（从Permission表同步）
await role.syncPermissions()

// 检查是否可删除
role.canDelete()

// 检查是否启用
role.isEnabled()

// 获取员工数量
await role.getEmployeeCount()
```

#### 静态方法

```javascript
// 根据代码查找
const role = await Role.findByCode('ADMIN')

// 根据名称查找
const role = await Role.findByName('系统管理员')

// 获取启用的角色
const roles = await Role.getActiveRoles()

// 获取系统角色
const systemRoles = await Role.getSystemRoles()

// 获取自定义角色
const customRoles = await Role.getCustomRoles()

// 按等级范围查询
const roles = await Role.getByLevelRange(50, 100)
```

### 2. Permission（权限模型）

**文件位置**：`models/Permission.js`  
**数据库表**：`permissions`

#### 字段说明

| 字段名 | 类型 | 必填 | 唯一 | 说明 |
|--------|------|------|------|------|
| id | INTEGER | ✅ | ✅ | 主键 |
| code | STRING(100) | ✅ | ✅ | 权限代码（如：client:read） |
| name | STRING(100) | ✅ | ❌ | 权限名称（如：查看客户） |
| module | STRING(50) | ✅ | ❌ | 所属模块 |
| action | STRING(50) | ✅ | ❌ | 权限动作 |
| description | TEXT | ❌ | ❌ | 权限描述 |
| group | STRING(50) | ❌ | ❌ | 权限分组 |
| isActive | BOOLEAN | ✅ | ❌ | 是否启用 |
| isSystem | BOOLEAN | ✅ | ❌ | 是否系统内置 |
| sortOrder | INTEGER | ✅ | ❌ | 排序顺序 |
| createdAt | DATE | ✅ | ❌ | 创建时间 |
| updatedAt | DATE | ✅ | ❌ | 更新时间 |

#### 权限代码格式

```
格式：module:action
示例：
  - client:read      (查看客户)
  - client:write     (编辑客户)
  - call:create      (创建外呼)
  - employee:delete  (删除员工)
  - *                (所有权限)
```

#### 模块列表

- `client` - 客户管理
- `call` - 外呼管理
- `employee` - 员工管理
- `role` - 角色管理
- `permission` - 权限管理
- `assignment` - 分配管理
- `report` - 报表管理
- `dashboard` - 仪表盘
- `system` - 系统管理

#### 动作列表

- `read` - 查看
- `create` - 创建
- `update` - 更新
- `delete` - 删除
- `export` - 导出
- `import` - 导入
- `manage` - 管理
- `view` - 查看
- `execute` - 执行

#### 实例方法

```javascript
// 检查是否可删除
permission.canDelete()

// 检查是否启用
permission.isEnabled()

// 获取角色数量
await permission.getRoleCount()

// 获取完整代码
permission.getFullCode()
```

#### 静态方法

```javascript
// 根据代码查找
const perm = await Permission.findByCode('client:read')

// 根据模块查找
const perms = await Permission.findByModule('client')

// 根据动作查找
const perms = await Permission.findByAction('read')

// 获取启用的权限
const perms = await Permission.getActivePermissions()

// 按模块分组
const grouped = await Permission.getGroupedByModule()

// 按分组获取
const grouped = await Permission.getGroupedByGroup()

// 批量创建
await Permission.bulkCreatePermissions([...])

// 查找或创建
const [perm, created] = await Permission.findOrCreateByModuleAction('client', 'read')
```

## 💡 使用示例

### 1. 创建角色

```javascript
const role = await Role.create({
  name: '区域经理',
  code: 'REGIONAL_MANAGER',
  description: '区域经理，管理区域内的销售团队',
  permissions: ['client:read', 'client:update', 'call:read'],
  level: 65,
  isActive: true,
  isSystem: false
});
```

### 2. 为角色添加权限

```javascript
const role = await Role.findByCode('SALES');

// 方法1：直接添加权限代码
await role.addPermissions(['client:export', 'call:export']);

// 方法2：通过关联表添加
const permission = await Permission.findByCode('report:view');
await role.addPermissionDetails(permission);

// 方法3：同步权限
await role.syncPermissions();
```

### 3. 检查权限

```javascript
const role = await Role.findByCode('SALES');

// 检查单个权限
if (role.hasPermission('client:read')) {
  console.log('有权限查看客户');
}

// 检查多个权限
const hasAllPerms = ['client:read', 'client:write'].every(
  perm => role.hasPermission(perm)
);
```

### 4. 查询角色及其权限

```javascript
const role = await Role.findByPk(1, {
  include: [{
    model: Permission,
    as: 'permissionDetails',
    through: { attributes: [] }
  }]
});

console.log(role.name);
console.log(role.permissionDetails); // 权限详情数组
```

### 5. 创建权限

```javascript
const permission = await Permission.create({
  code: 'invoice:create',
  name: '创建发票',
  module: 'invoice',
  action: 'create',
  description: '创建新的发票',
  group: '发票管理',
  isActive: true,
  isSystem: false,
  sortOrder: 100
});
```

### 6. 按模块获取权限

```javascript
// 获取客户模块的所有权限
const clientPerms = await Permission.findByModule('client');

// 按模块分组获取所有权限
const grouped = await Permission.getGroupedByModule();
console.log(grouped.client); // 客户模块权限数组
console.log(grouped.call);   // 外呼模块权限数组
```

### 7. 员工权限检查

```javascript
const employee = await Employee.findByPk(1, {
  include: [{
    model: Role,
    as: 'role',
    include: [{
      model: Permission,
      as: 'permissionDetails'
    }]
  }]
});

// 检查员工是否有权限
if (employee.role && employee.role.hasPermission('client:write')) {
  // 允许编辑客户
}

// 或者检查员工自己的权限数组
if (employee.hasPermission('client:write')) {
  // 允许编辑客户
}
```

### 8. 权限中间件示例

```javascript
// middleware/checkPermission.js
const checkPermission = (requiredPermission) => {
  return async (req, res, next) => {
    try {
      const employee = await Employee.findByPk(req.user.id, {
        include: [{ model: Role, as: 'role' }]
      });

      if (!employee || !employee.role) {
        return res.status(403).json({ error: '无权限访问' });
      }

      // 检查角色权限
      if (employee.role.hasPermission(requiredPermission)) {
        return next();
      }

      // 检查员工个人权限
      if (employee.hasPermission(requiredPermission)) {
        return next();
      }

      return res.status(403).json({ error: '权限不足' });
    } catch (error) {
      return res.status(500).json({ error: '权限检查失败' });
    }
  };
};

// 使用示例
router.post('/clients', 
  authenticate, 
  checkPermission('client:create'), 
  createClient
);
```

## 🔐 权限检查流程

```
1. 用户请求 → 2. 身份验证 → 3. 获取用户角色 → 4. 检查权限
                                                    ↓
                                              5. 允许/拒绝
```

### 权限检查优先级

1. **超级权限** (`*`) - 拥有所有权限
2. **角色权限** - 角色的permissions数组
3. **员工个人权限** - 员工的permissions数组
4. **关联表权限** - role_permissions表中的权限

## 📊 索引设计

### roles表索引
- `idx_role_name` - 角色名称（唯一）
- `idx_role_code` - 角色代码（唯一）
- `idx_role_active` - 是否启用
- `idx_role_level` - 角色等级
- `idx_role_system` - 是否系统角色

### permissions表索引
- `idx_permission_code` - 权限代码（唯一）
- `idx_permission_module` - 模块
- `idx_permission_action` - 动作
- `idx_permission_active` - 是否启用
- `idx_permission_system` - 是否系统权限
- `idx_permission_group` - 权限分组
- `idx_permission_sort` - 排序
- `idx_permission_module_action` - 模块+动作（复合）

### role_permissions表索引
- `idx_rp_role_id` - 角色ID
- `idx_rp_permission_id` - 权限ID
- `idx_rp_role_permission` - 角色ID+权限ID（唯一）

## ⚠️ 注意事项

### 1. 系统角色保护
- 系统内置角色（`isSystem: true`）不可删除
- 修改系统角色需谨慎

### 2. 权限代码规范
- 格式：`module:action`
- 只能包含小写字母、下划线和冒号
- 建议使用统一的命名规范

### 3. 角色等级
- 等级范围：1-100
- 数字越大权限越高
- 建议预留等级空间便于扩展

### 4. 权限检查性能
- 使用JSON字段快速检查权限
- 定期同步permissions字段
- 考虑使用Redis缓存权限

### 5. 数据一致性
- 删除角色时员工的roleId会被设置为NULL
- 删除权限时会自动删除关联关系
- 建议定期检查数据一致性
