# 🌐 浏览器访问指南

## 快速启动服务器

### 方法1：使用命令行启动（推荐）

1. **打开PowerShell或命令提示符**

2. **进入项目目录**
   ```powershell
   cd tax-crm-system
   ```

3. **启动服务器**
   ```powershell
   npm run dev
   ```

4. **看到以下信息表示启动成功：**
   ```
   🚀 服务器启动成功！
   📍 服务地址: http://localhost:3000
   ```

### 方法2：如果遇到问题

如果服务器启动失败，请先检查：

```powershell
# 1. 检查环境变量文件是否存在
dir .env

# 2. 如果不存在，创建它
copy .env.example .env

# 3. 编辑 .env 文件，填入数据库配置

# 4. 确保PostgreSQL正在运行
# 5. 确保数据库已创建
```

## 🌐 在浏览器中访问

### 1. 健康检查页面

启动服务器后，在浏览器地址栏输入：

```
http://localhost:3000/health
```

**预期结果：**
```json
{
  "success": true,
  "message": "服务运行正常",
  "timestamp": "2024-01-01T00:00:00.000Z"
}
```

## 🛠️ 使用浏览器测试API

### 方法1：使用浏览器开发者工具

1. 打开浏览器（Chrome/Firefox/Edge）
2. 按 `F12` 打开开发者工具
3. 切换到 **Network（网络）** 标签
4. 访问 `http://localhost:3000/health`

### 方法2：使用Postman或类似工具

推荐使用 **Postman** 或 **Thunder Client**（VS Code扩展）来测试API。

### 方法3：使用PowerShell测试

```powershell
# 测试健康检查
Invoke-RestMethod -Uri "http://localhost:3000/health"

# 注册用户
$body = @{
    username = "admin"
    email = "admin@test.com"
    password = "admin123"
    role = "admin"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:3000/api/auth/register" `
    -Method Post `
    -Body $body `
    -ContentType "application/json"
```

## ⚠️ 如果浏览器显示 ERR_CONNECTION_REFUSED

这表示服务器没有运行。请：

1. **检查服务器是否启动**
   - 查看命令行窗口，确认服务器正在运行
   - 应该看到 "服务器运行在端口 3000"

2. **检查端口是否被占用**
   ```powershell
   netstat -ano | findstr :3000
   ```

3. **检查防火墙设置**
   - 确保允许本地3000端口访问

4. **尝试其他端口**
   - 修改 `.env` 文件中的 `PORT=3001`
   - 然后访问 `http://localhost:3001/health`
