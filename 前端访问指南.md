## 📱 员工管理功能详解

### 员工列表页面 (`/employees`)

**功能特性：**
- ✅ 搜索员工（按姓名、工号、邮箱）
- ✅ 多条件筛选（部门、职位、状态、角色）
- ✅ 分页显示
- ✅ 快速操作（查看、编辑、删除）
- ✅ 批量操作
- ✅ 导出数据

**操作步骤：**
1. 点击左侧菜单 "员工管理"
2. 使用搜索框快速查找员工
3. 点击 "新增员工" 按钮添加新员工
4. 点击操作列的图标进行编辑或删除

### 员工详情页面 (`/employees/:id`)

**功能特性：**
- ✅ 基本信息展示
- ✅ 工作统计（客户数、通话数、成功率）
- ✅ 工作趋势图表
- ✅ 最近活动记录
- ✅ 权限管理

**访问方式：**
- 在员工列表点击 "查看" 图标
- 或直接访问 `/employees/1`（员工ID）

### 员工表单页面 (`/employees/new` 或 `/employees/edit/:id`)

**功能特性：**
- ✅ 完整的表单验证
- ✅ 头像上传
- ✅ 角色选择
- ✅ 权限配置
- ✅ 自动生成工号
- ✅ 密码自动生成（新增时）

**表单字段：**
- 基本信息：姓名、工号、用户名、邮箱、手机
- 工作信息：部门、职位、入职日期
- 权限信息：角色、访问等级、特殊权限
- 头像上传

### 角色管理页面 (`/roles`)

**功能特性：**
- ✅ 角色列表管理
- ✅ 权限树形选择
- ✅ 角色创建/编辑/删除
- ✅ 权限批量分配
- ✅ 系统角色保护

**操作步骤：**
1. 访问 `/roles` 或通过员工管理进入
2. 点击 "新增角色" 创建新角色
3. 在权限树中勾选需要的权限
4. 保存角色配置

## ❓ 常见问题

### 1. 页面无法访问？
- 确认前端服务是否启动：`npm run dev`（在 client 目录）
- 确认后端服务是否启动：`npm run dev`（在根目录）
- 检查端口是否被占用

### 2. 登录失败？
- 检查后端服务是否正常运行
- 确认数据库连接正常
- 查看浏览器控制台的错误信息

### 3. 员工管理页面空白？
- 检查是否已登录
- 确认用户角色有访问权限（需要 admin 或 manager 角色）
- 查看控制台是否有 API 错误

### 4. 数据不显示？
- 确认数据库中有数据
- 运行种子数据：`npm run db:seed`
- 检查 API 响应是否正常
