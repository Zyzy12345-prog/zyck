# 数据库迁移故障排除指南

## ❌ 常见错误及解决方案

### 错误1：语法错误 在 "USING" 或附近的

**错误信息：**
```
ERROR: 语法错误 在 "USING" 或附近的
```

**原因：**
PostgreSQL 修改 ENUM 类型字段时语法不正确

**解决方案：**
✅ 已修复！新的迁移文件使用正确的方法：
1. 创建新列
2. 迁移数据
3. 删除旧列
4. 重命名新列

**执行修复：**
```bash
# 1. 如果迁移失败，先回滚
npx sequelize-cli db:migrate:undo

# 2. 重新执行迁移（使用修复后的文件）
npx sequelize-cli db:migrate
```

---

### 错误2：ENUM 类型已存在

**错误信息：**
```
ERROR: type "enum_call_records_call_type" already exists
```

**解决方案：**
```sql
-- 连接数据库
psql -U postgres -d tax_crm

-- 删除旧的 ENUM 类型
DROP TYPE IF EXISTS "enum_call_records_call_type" CASCADE;
DROP TYPE IF EXISTS "enum_call_records_status" CASCADE;
DROP TYPE IF EXISTS "enum_call_records_direction" CASCADE;
DROP TYPE IF EXISTS "enum_call_records_provider" CASCADE;

-- 退出
\q

-- 重新运行迁移
npx sequelize-cli db:migrate
```

---

### 错误3：phone_number 包含 NULL 值

**错误信息：**
```
ERROR: column "phone_number" contains null values
```

**解决方案：**
```sql
-- 手动填充 phone_number
psql -U postgres -d tax_crm

UPDATE call_records cr
SET phone_number = COALESCE(c.phone, '未知号码')
FROM clients c
WHERE cr.client_id = c.id AND cr.phone_number IS NULL;

UPDATE call_records
SET phone_number = '未知号码'
WHERE phone_number IS NULL;

\q

-- 重新运行迁移
npx sequelize-cli db:migrate
```

---

### 错误4：迁移卡住不动

**原因：**
表被锁定或有其他连接占用

**解决方案：**
```sql
-- 查看活动连接
psql -U postgres -d tax_crm

SELECT pid, usename, application_name, state, query
FROM pg_stat_activity
WHERE datname = 'tax_crm';

-- 终止其他连接（替换 PID）
SELECT pg_terminate_backend(PID);

\q

-- 重新运行迁移
npx sequelize-cli db:migrate
```

---

### 错误5：权限不足

**错误信息：**
```
ERROR: permission denied for table call_records
```

**解决方案：**
```sql
-- 授予权限
psql -U postgres -d tax_crm

GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO postgres;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO postgres;

\q
```

## 🔧 完全重置方案

如果迁移完全失败，可以重置数据库：

### 方案1：删除并重建表（⚠️ 会丢失数据）

```bash
# 1. 备份数据
pg_dump -U postgres -d tax_crm > backup.sql

# 2. 连接数据库
psql -U postgres -d tax_crm

# 3. 删除外呼记录表
DROP TABLE IF EXISTS call_records CASCADE;

# 4. 删除所有 ENUM 类型
DROP TYPE IF EXISTS "enum_call_records_call_type" CASCADE;
DROP TYPE IF EXISTS "enum_call_records_status" CASCADE;
DROP TYPE IF EXISTS "enum_call_records_direction" CASCADE;
DROP TYPE IF EXISTS "enum_call_records_provider" CASCADE;

# 5. 退出
\q

# 6. 重新运行所有迁移
npx sequelize-cli db:migrate:undo:all
npx sequelize-cli db:migrate
```

### 方案2：重建整个数据库（⚠️ 会丢失所有数据）

```bash
# 1. 备份数据库
pg_dump -U postgres -d tax_crm > backup_full.sql

# 2. 删除数据库
psql -U postgres -c "DROP DATABASE tax_crm;"

# 3. 创建数据库
psql -U postgres -c "CREATE DATABASE tax_crm;"

# 4. 运行所有迁移
cd d:/tax-crm-system
npx sequelize-cli db:migrate
```
