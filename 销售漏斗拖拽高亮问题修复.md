# 销售漏斗拖拽高亮问题修复

## 🔧 问题诊断

### 用户反馈
"为什么我放到意向中，意向不显示高亮"

### 问题原因
1. **碰撞检测过于严格** - 只检测中心点，导致卡片边缘进入时不触发
2. **可放置区域太小** - 阶段宽度和内边距不够大
3. **检测优先级错误** - 中心点检测优先，但用户拖拽时卡片可能还没完全进入

---

## ✅ 解决方案

### 1. 优化碰撞检测算法

**改进前：**
```javascript
// 只检测中心点
const centerCollisions = closestCenter(args);
```

**改进后：**
```javascript
// 优先使用矩形相交检测
const rectCollisions = rectIntersection(args);
if (rectCollisions.length > 0) {
  return rectCollisions; // 只要有重叠就触发
}

// 兜底使用中心点检测
return closestCenter(args);
```

**优势：**
- ✅ 只要拖拽卡片与目标阶段有**任何重叠**就高亮
- ✅ 不需要等到中心点进入
- ✅ 更灵敏，更容易触发

---

### 2. 增大可放置区域

#### 阶段宽度
- 改进前：340px
- **改进后：360px** ⬆️ +20px

#### 阶段间距
- 改进前：8px
- **改进后：6px** ⬇️ -2px（减少间距，增加可用空间）

#### 列表内边距
- 改进前：8px
- **改进后：12px** ⬆️ +4px

#### 拖拽时内边距
- 改进前：12px
- **改进后：16px** ⬆️ +4px

---

### 3. 检测优先级调整

**新的检测顺序：**
1. **矩形相交检测**（主要） - 只要有重叠就触发
2. **中心点检测**（兜底） - 处理边缘情况
