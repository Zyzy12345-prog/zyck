## 🐛 常见问题解决

### 问题1: 数据库迁移编码错误

**症状**：
```
错误: 编码"GBK"的字符0x0xa8 0x27在编码"UTF8"没有相对应值
```

**解决方案**：
使用 `migration-fix.sql` 而不是 `complete-customer-detail-migration.sql`

```powershell
psql -U postgres -d tax_crm -f migration-fix.sql
```

### 问题2: react-quill 未安装

**症状**：
```
Failed to resolve import "react-quill"
```

**解决方案**：
```powershell
cd client
npm install react-quill
```

### 问题3: 表已存在错误

**症状**：
```
错误: 关系 "follow_ups" 已经存在
```

**解决方案**：
这是正常的，SQL 使用了 `IF NOT EXISTS`，会自动跳过已存在的表。

### 问题4: 外键约束错误

**症状**：
```
错误: 关系 "clients" 不存在
```

**解决方案**：
确保 clients 和 users 表已经存在。检查之前的迁移是否成功。
