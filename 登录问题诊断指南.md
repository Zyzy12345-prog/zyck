# 登录问题诊断指南

## 问题描述
邮箱和密码都正确，但登录时显示"请检测"错误信息。

## 可能的原因

### 1. 后端服务未启动 ⚠️
**症状**: 前端无法连接到后端API

**检查方法**:
```bash
# 检查后端是否在运行
curl http://localhost:3000/health
# 或在浏览器访问: http://localhost:3000/health
```

**解决方法**:
```bash
# 启动后端服务
cd d:\tax-crm-system
npm start
```

### 2. 数据库连接问题 ⚠️
**症状**: 后端启动但无法查询数据库

**检查方法**:
```bash
# 运行数据库检查脚本
node check-users.js
```

**解决方法**:
- 确保 PostgreSQL 服务正在运行
- 检查 `.env` 文件中的数据库配置
- 运行: `node init-database.js`

### 3. CORS 跨域问题 ⚠️
**症状**: 浏览器控制台显示 CORS 错误

**检查方法**:
- 打开浏览器开发者工具 (F12)
- 查看 Console 标签页
- 查看 Network 标签页，检查请求状态

**解决方法**:
检查 `app.js` 或 `server.js` 中的 CORS 配置

### 4. API 路径错误 ⚠️
**症状**: 404 Not Found 错误

**检查方法**:
```bash
# 测试登录接口
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com","password":"admin123"}'
```

### 5. 密码验证失败 ⚠️
**症状**: 401 Unauthorized 错误

**可能原因**:
- 密码不正确
- 密码加密方式不匹配
- bcrypt 版本问题

## 快速诊断步骤

### 第1步: 检查后端服务状态

```bash
# 方法1: 检查进程
netstat -ano | findstr :3000

# 方法2: 访问健康检查接口
# 在浏览器打开: http://localhost:3000/health
```

**预期结果**: 应该看到后端服务正在监听 3000 端口

### 第2步: 检查浏览器控制台

1. 打开浏览器开发者工具 (F12)
2. 切换到 **Console** 标签页
3. 尝试登录
4. 查看是否有错误信息

**常见错误**:
- `Network Error` - 后端未启动
- `CORS Error` - 跨域配置问题
- `401 Unauthorized` - 密码错误
- `500 Internal Server Error` - 后端代码错误

### 第3步: 检查 Network 请求

1. 打开浏览器开发者工具 (F12)
2. 切换到 **Network** 标签页
3. 尝试登录
4. 找到 `login` 请求
5. 查看请求详情:
   - **Request URL**: 应该是 `http://localhost:3000/api/auth/login`
   - **Status Code**: 
     - 200 = 成功
     - 401 = 密码错误
     - 404 = 路径错误
     - 500 = 服务器错误
   - **Response**: 查看具体错误信息

### 第4步: 检查用户数据

```bash
# 运行用户检查脚本
node check-users.js
```

**检查项**:
- ✅ users 表存在
- ✅ 用户数据存在
- ✅ 密码已加密 (bcrypt)
- ✅ 用户状态为 active

### 第5步: 测试登录接口


### 解决方案 3: 重置用户密码

```bash
# 创建密码重置脚本
node reset-password.js
```

创建 `reset-password.js`:
```javascript
const { Sequelize } = require('sequelize');
const bcrypt = require('bcryptjs');
require('dotenv').config();

const sequelize = new Sequelize(
  process.env.DB_NAME || 'tax_crm',
  process.env.DB_USER || 'postgres',
  process.env.DB_PASSWORD || '',
  {
    host: process.env.DB_HOST || 'localhost',
    port: process.env.DB_PORT || 5432,
    dialect: 'postgres',
    logging: false
  }
);

async function resetPassword() {
  try {
    const email = 'admin@example.com';
    const newPassword = 'admin123';
    
    const hashedPassword = await bcrypt.hash(newPassword, 10);
    
    await sequelize.query(`
      UPDATE users 
      SET password = :password, 
          "updatedAt" = NOW()
      WHERE email = :email
    `, {
      replacements: { email, password: hashedPassword }
    });
    
    console.log(`✅ 密码已重置: ${email} -> ${newPassword}`);
  } catch (error) {
    console.error('❌ 重置失败:', error.message);
  } finally {
    await sequelize.close();
    process.exit(0);
  }
}

resetPassword();
```

## 详细错误信息对照表

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| "请检查邮箱和密码" | 密码验证失败 | 确认密码正确，或重置密码 |
| "账户已被禁用" | 用户状态不是 active | 更新用户状态为 active |
| "Network Error" | 后端未启动 | 启动后端服务 |
| "CORS Error" | 跨域配置问题 | 检查 CORS 配置 |
| "500 Internal Server Error" | 后端代码错误 | 查看后端日志 |

## 常见问题 FAQ

### Q: 后端显示"数据库连接成功"但登录失败？
A: 检查用户表数据和密码加密，运行 `node check-users.js`

### Q: 浏览器显示"Network Error"？
A: 后端服务未启动或端口被占用，运行 `npm start`

### Q: 密码确定正确但还是登录失败？
A: 可能是密码加密问题，运行密码重置脚本

### Q: 之前能登录，现在突然不能了？
A: 可能是数据库表结构变化，运行 `node init-database.js` 检查
