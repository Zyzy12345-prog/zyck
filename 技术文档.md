## 项目概述

**项目名称**: 财税CRM系统  
**项目类型**: 企业级客户关系管理系统  
**开发模式**: 前后端分离  
**架构模式**: RESTful API + WebSocket  

### 系统定位
面向财税服务行业的专业CRM系统，提供客户管理、销售漏斗、外呼管理、数据分析等全方位功能。

## 数据库设计

### 数据库选型：PostgreSQL

#### 核心优势
1. **开源免费**: 无许可费用，降低成本
2. **ACID事务**: 保证数据一致性
3. **JSON支持**: 灵活存储非结构化数据
4. **全文搜索**: 内置搜索引擎
5. **扩展性**: 支持分区、复制、集群
6. **可靠性**: 企业级稳定性

### 数据模型设计

#### 核心表结构（32张表）

##### 1. 用户与权限模块
- **users**: 用户表（登录账号）
- **roles**: 角色表（admin、manager、sales、operator）
- **permissions**: 权限表（细粒度权限控制）
- **departments**: 部门表（组织架构）
- **employees**: 员工表（员工详细信息）

##### 2. 客户管理模块
- **clients**: 客户表（核心业务数据）
- **customer_leads**: 客户线索表（潜在客户）
- **customer_pool**: 客户公海池（未分配客户）
- **customer_tags**: 客户标签表（标签定义）
- **client_tag_relations**: 客户标签关联表（多对多）
- **lead_tags**: 线索标签表
- **industry_categories**: 行业分类表（三级分类）

##### 3. 跟进与沟通模块
- **follow_ups**: 跟进记录表
- **follow_up_comments**: 跟进评论表
- **follow_up_reminders**: 跟进提醒表
- **lead_follow_ups**: 线索跟进表
- **customer_discussions**: 内部讨论表
- **communication_records**: 通讯记录表（电话、短信、邮件）

##### 4. 销售漏斗模块
- **sales_opportunities**: 销售商机表
- **sales_stages**: 销售阶段表（可自定义）
- **opportunity_stage_histories**: 阶段变更历史表

##### 5. 外呼模块
- **call_records**: 外呼记录表
- **call_tasks**: 外呼任务表
- **call_script_templates**: 话术模板表

##### 6. 客户评分模块
- **client_scores**: 客户评分表
- **client_value_analyses**: 客户价值分析表

##### 7. 文件管理模块
- **customer_files**: 客户文件表
- **file_uploads**: 文件上传记录表

##### 8. 实时聊天模块
- **chat_rooms**: 聊天室表
- **chat_messages**: 聊天消息表
- **chat_participants**: 聊天参与者表

##### 9. 任务分配模块
- **assignments**: 任务分配表

### 数据库特性应用

#### 1. 外键约束
```sql
-- 保证数据完整性
FOREIGN KEY (client_id) REFERENCES clients(id) ON DELETE CASCADE
```

#### 2. 索引优化
```sql
-- 提升查询性能
CREATE INDEX idx_clients_company_name ON clients(company_name);
CREATE INDEX idx_follow_ups_client_id ON follow_ups(client_id);
```

#### 3. JSON字段
```sql
-- 灵活存储扩展数据
metadata JSONB DEFAULT '{}'
```

#### 4. 枚举类型
```sql
-- 限制字段取值范围
status VARCHAR(20) CHECK (status IN ('active', 'inactive', 'pending'))
```

#### 5. 时间戳
```sql
-- 自动记录创建和更新时间
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
```

|------|------|

## 总结

### 技术栈优势

1. **全栈JavaScript**: 降低学习成本，提高开发效率
2. **现代化框架**: React 18 + Vite，开发体验极佳
3. **企业级组件**: Ant Design，快速构建专业界面
4. **强大数据库**: PostgreSQL，保证数据安全可靠
5. **实时通信**: Socket.io，支持即时消息推送
6. **可扩展性**: 模块化设计，易于维护和扩展

### 适用场景

- ✅ 中小型企业CRM系统
- ✅ 财税服务行业管理系统
- ✅ 客户关系管理平台
- ✅ 销售团队协作工具
- ✅ 数据分析与报表系统

### 未来扩展

- 📱 移动端适配（React Native）
- 🤖 AI智能推荐（客户评分优化）
- 📊 BI商业智能（深度数据分析）
- 🔗 第三方集成（企业微信、钉钉）
- ☁️ 云原生部署（Docker + Kubernetes）
