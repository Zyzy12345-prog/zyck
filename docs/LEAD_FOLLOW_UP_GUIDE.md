# 线索跟进记录系统 - 功能说明

## 📋 功能概述

线索跟进记录系统是客户拓展模块的重要组成部分，用于记录和管理销售人员对潜在客户的跟进活动。

---

## ✨ 核心功能

### 1. 跟进记录管理

#### 创建跟进记录
- 记录跟进时间
- 选择跟进方式（电话、邮件、拜访、微信、其他）
- 填写详细的跟进内容
- 评估跟进结果（积极、中性、消极、未响应）
- 记录跟进时长
- 计划下次跟进时间和内容

#### 查看跟进记录
- 时间线展示所有跟进历史
- 按跟进方式筛选
- 按跟进结果筛选
- 分页浏览

#### 编辑跟进记录
- 修改跟进内容
- 更新跟进结果
- 调整下次跟进计划
- 权限控制：仅创建人和管理员可编辑

#### 删除跟进记录
- 删除不需要的记录
- 权限控制：仅创建人和管理员可删除

---

### 2. 跟进统计

#### 统计指标
- **总跟进次数**: 该线索的累计跟进次数
- **最近跟进**: 最后一次跟进的时间
- **下次跟进**: 计划的下次跟进时间
- **积极反馈**: 获得积极反馈的次数

#### 按维度统计
- **按跟进方式**: 电话、邮件、拜访等各方式的次数
- **按跟进结果**: 积极、中性、消极、未响应的分布

---

### 3. 待跟进提醒

#### 功能特点
- 自动识别到期的跟进计划
- 按时间排序，优先显示紧急的
- 显示线索基本信息
- 快速跳转到跟进页面

---

## 🎯 使用场景

### 场景1：首次联系新线索
1. 销售人员获得新线索
2. 进行首次电话沟通
3. 创建跟进记录：
   - 跟进方式：电话
   - 跟进内容：介绍公司和产品，了解客户需求
   - 跟进结果：积极
   - 下次跟进：3天后发送详细资料

### 场景2：持续跟进
1. 查看待跟进列表
2. 找到需要跟进的线索
3. 进行跟进活动
4. 记录本次跟进：
   - 跟进方式：邮件
   - 跟进内容：发送产品资料和报价单
   - 跟进结果：中性
   - 下次跟进：1周后电话确认

### 场景3：拜访客户
1. 预约客户拜访
2. 上门拜访并演示产品
3. 记录拜访详情：
   - 跟进方式：拜访
   - 跟进内容：详细演示产品功能，客户提出具体需求
   - 跟进时长：120分钟
   - 跟进结果：积极
   - 下次跟进：3天后提交方案

## 📊 数据模型

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| lead_id | INTEGER | 线索ID |
| follow_up_type | ENUM | 跟进方式 |
| follow_up_date | TIMESTAMP | 跟进时间 |
| content | TEXT | 跟进内容 |
| result | ENUM | 跟进结果 |
| next_follow_up_date | TIMESTAMP | 下次跟进时间 |
| next_follow_up_plan | TEXT | 下次跟进计划 |
| duration | INTEGER | 跟进时长（分钟） |
| attachments | JSONB | 附件列表 |
| tags | TEXT[] | 标签 |
| is_important | BOOLEAN | 是否重要 |
| created_by | INTEGER | 创建人 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 枚举值

**跟进方式 (follow_up_type)**:
- `phone`: 电话
- `email`: 邮件
- `visit`: 拜访
- `wechat`: 微信
- `other`: 其他

**跟进结果 (result)**:
- `positive`: 积极
- `neutral`: 中性
- `negative`: 消极
- `no_response`: 未响应

## 🔌 API接口

### 获取线索的跟进记录列表
```
GET /api/leads/:leadId/follow-ups
Query参数:
  - page: 页码
  - limit: 每页数量
  - followUpType: 跟进方式筛选
  - result: 结果筛选
```

### 获取线索的跟进统计
```
GET /api/leads/:leadId/follow-ups/statistics
```

### 创建跟进记录
```
POST /api/leads/:leadId/follow-ups
Body: {
  followUpType: string (必填)
  followUpDate: datetime (必填)
  content: string (必填)
  result: string
  duration: number
  nextFollowUpDate: datetime
  nextFollowUpPlan: string
}
```

### 获取单个跟进记录
```
GET /api/follow-ups/:id
```

### 更新跟进记录
```
PUT /api/follow-ups/:id
Body: 同创建接口
```

### 删除跟进记录
```
DELETE /api/follow-ups/:id
```

### 获取待跟进列表
```
GET /api/follow-ups/pending
Query参数:
  - page: 页码
  - limit: 每页数量
```

## 💡 最佳实践

### 1. 及时记录
- 每次跟进后立即记录
- 趁记忆清晰时填写详细内容
- 不要拖延，避免遗忘重要信息

### 2. 详细描述
- 记录客户的具体需求
- 记录客户的疑虑和问题
- 记录承诺的事项
- 记录客户的反馈

### 3. 合理规划
- 根据客户意向设置下次跟进时间
- 高意向客户缩短跟进周期
- 低意向客户适当延长周期
- 制定具体的跟进计划

### 4. 结果评估
- 客观评估跟进效果
- 积极：客户明确表示兴趣，愿意继续沟通
- 中性：客户态度一般，需要继续观察
- 消极：客户明确拒绝或不感兴趣
- 未响应：客户未接听或未回复

### 5. 数据分析
- 定期查看跟进统计
- 分析哪种跟进方式最有效
- 总结成功经验
- 改进跟进策略

## 🔒 权限控制

### 查看权限
- 销售人员：只能查看自己负责的线索的跟进记录
- 经理/管理员：可以查看所有跟进记录

### 创建权限
- 所有角色都可以创建跟进记录

### 编辑权限
- 创建人可以编辑自己的记录
- 管理员可以编辑所有记录

### 删除权限
- 创建人可以删除自己的记录
- 管理员可以删除所有记录
