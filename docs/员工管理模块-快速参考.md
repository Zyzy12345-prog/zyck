## 📊 数据库表结构概览

### 1. departments（部门表）
- **记录数：** 6个初始部门
- **主要字段：** id, name, code, parent_id, manager_id, level
- **特性：** 树形结构，支持多级部门

### 2. employees（员工表）
- **记录数：** 3个初始员工
- **主要字段：** id, employee_no, name, username, password, department_id
- **特性：** 密码加密，自动生成工号

### 3. roles（角色表）
- **记录数：** 5个系统角色
- **主要字段：** id, name, code, permissions
- **特性：** JSONB存储权限，系统角色不可删除

### 4. employee_roles（员工角色关联表）
- **记录数：** 3条关联记录
- **主要字段：** employee_id, role_id, assigned_by
- **特性：** 多对多关系，防止重复分配

## 🔧 常用操作

### 查看迁移状态
```bash
npx sequelize-cli db:migrate:status
```

### 回滚迁移
```bash
# 回滚最后一次
npx sequelize-cli db:migrate:undo

# 回滚所有
npx sequelize-cli db:migrate:undo:all
```

### 重新初始化
```bash
# 回滚所有迁移
npx sequelize-cli db:migrate:undo:all

# 重新运行迁移
npx sequelize-cli db:migrate

# 重新初始化数据
node scripts/init-employee-data.js
```

## 📝 模型使用示例

### 创建部门
```javascript
const department = await Department.create({
  name: '研发部',
  code: 'RD_DEPT',
  level: 1,
  status: 'active'
});
```

### 创建员工
```javascript
const employee = await Employee.create({
  name: '张三',
  username: 'zhangsan',
  password: 'password123',
  departmentId: department.id,
  status: 'active'
});
```

### 分配角色
```javascript
await EmployeeRole.create({
  employeeId: employee.id,
  roleId: role.id
});
```

### 查询员工（含部门和角色）
```javascript
const employee = await Employee.findByPk(id, {
  include: [
    { model: Department, as: 'department' },
    { model: Role, as: 'roles' }
  ]
});
```

### 检查权限
```javascript
const hasPermission = await employee.hasPermission('client.create');
```

## ⚠️ 重要提示

1. **首次运行必须按顺序执行：**
   - 先运行迁移
   - 再初始化数据

2. **密码安全：**
   - 生产环境请修改默认密码
   - 密码自动使用bcrypt加密

3. **系统角色：**
   - 不可删除
   - 编码不可修改
   - 权限可以更新

4. **工号生成：**
   - 格式：YYYYMM0001
   - 自动递增
   - 按月重置

5. **软删除：**
   - 使用status字段
   - 不进行物理删除
   - 保留历史数据

## 🐛 常见问题

### Q1: 迁移失败怎么办？
**A:** 检查数据库连接配置，确保PostgreSQL服务正在运行。

### Q2: 初始化数据重复怎么办？
**A:** 脚本使用`findOrCreate`，重复运行不会创建重复数据。

### Q3: 忘记管理员密码怎么办？
**A:** 运行初始化脚本会重置为默认密码`admin123456`。

### Q4: 如何添加新的权限？
**A:** 在角色的permissions字段中添加新的权限键值对。

### Q5: 如何创建子部门？
**A:** 设置`parentId`为父部门的ID，`level`会自动计算。
